# Decision Log

This file records important project choices so the repo does not drift.

## 2026-07-01 — Start clean

Decision: create a fresh BBXAI repo instead of forking an older project.

Reason: older repos contain useful patterns but also old assumptions, naming, and workflow baggage.

## 2026-07-01 — Emotional acceptance test

Decision: the first accepted version is defined by the wow moment, not a minimum checklist.

Acceptance line:

> I record my own sounds, BBXAI arranges them, and it feels like I made music.

## 2026-07-01 — Human source material

Decision: BBXAI should preserve the user as the creative source.

The AI can assist with arrangement, placement, timing, balance, and production polish, but the recorded sounds and final taste stay human.

## 2026-07-01 — Engine before interface

Decision: prove the audio/arrangement feeling before building a full mobile interface.

Reason: a clean app shell is not useful if the engine cannot create the wow moment.
